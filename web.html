WEB

  1. 브라우저 주소창에 도메인을 입력한 후 웹페이지가 보이기 까지의 과정

    URL을 입력하면 서버에 요청이 전송된다. 해당 페이지에 있는 여러 리소스를 전송받고, 브라우저는 해당 리소스가 담긴 html과 css를 W3C명세에 따라 렌더링 엔진을 통해 해석한다.
    렌더링 엔진은 먼저 HTML을 파싱하고 DOM트리를 구축한다. 이후 CSS파싱을 시작해 CSS정보를 스타일 구조체로 생성한다. 이 두 가지를 연결시켜 렌더트리를 만들고 렌더 트리를 통해 문서가 리소스를 포함한 구성된 상태가 된다.

  2. RESTful API

    REpresentational State Transfer는 어떤 자원에 대해 CRUD(Create, Read, Update, Delete)연산을 수행하기 위해 URI형태로 요청을 보내는 것이다. 주로 사용되는 Method로는 Get(Read), Post(Create), Put(Update), Delete(Delete)가 있다.

  3. 프로토콜

    프로토콜은 정보통신을 위해 한 시스템이 다른 시스템과 통신을 원활하게 수행할 수 있도록 해주는 통신 규약
    컴퓨터 간 또는 단맑기 간 에러 없이 효율적인 정보를 주고받기 위해 설정한 통신 규칙이다.

  4. HTTP와 HTTPS

    HTTPS(HyperText Transfer Protocol Secure)는 기존 프로토콜(컴퓨터 사이의 데이터 교환 방식 규칙 정의)에서 데이터가 쉽게 도난당할 수 있는 것을 방지하기 위해서 SSL(보안 소켓)을 사용하면서 브라우저와 서버 사이에 안전하게 암호화된 연결을 만들 수 있게 도와준다.
    또한, TLS(전송 계층 보안) 프로토콜을 통해서도 보안을 유지한다. TLS는 데이터 무결성을 제공하므로 데이터가 전송 중에 손상되거나 수정되는 것을 방지한다.
    HTTPS의 장점은 검색엔진 최적화로 구글과 같은 포털에서는 HTTPS를 사용하는 웹사이트에 가산점 부여, HTTPS가 아니라면 주의 요함과 같은 경고 문구

  5. 캐시(Cache)

    캐시는 데이터나 값을 미리 복사해 놓은 임시 저장소, 캐시는 데이터를 접근하는 시간이 오래 걸리는 경우나 값을 계산하는 시간이 오래 걸리는 로직 등에 사용한다. 캐시가 없다면 데이터가 변경되지 않는 고정 데이터임에도 계속해서 네트워크를 통해 데이터를 다운 받아야 한다. 불필요한 데이터를 계속 받는다면 데이터 비용의 비중도 올라가게 된다. 그리고 결국 사용자 측면에서도 로딩 속도가 반복해서 느려지게 되고, 이는 사용자에게 좋지 못한 경험을 제공한다. 캐시를 사용하면 새로고침 시에 반복되는 데이터가 네트워크를 통하지 않으므로 네트워크 사용량이 줄고, 브라우저 로딩 속도가 빨라지게 되며, 이는 빠른 사용자 경험으로 직결된다.

  6. 쿠키

    쿠키는 만료 기간이 있는 Key, Value 형태의 저장소를 말한다. 대부분의 브라우저가 지원하며, HTTP 요청마다 포함되어 API 호출의 부담으로 작용할 수 있다. 용량은 약 4KB정도로 작은 편이다. 쿠키 자체의 도난 위험이 있어 최근에는 구글 크롬의 샌드박스 정책과 같이 보안의 위험이 있는 경우를 방지하기 위해 노력하는 중이다.
    만료기간을 두고 몇일 보지 않는 팝업 같은 부분은 쿠키를 사용하는 것이 좋다.

  7. 로컬스토리지, 세션스토리지

    로컬스토리지의 가장 큰 특징은 브라우저 창을 닫더라도 데이터가 유지된다. 장점은 서버에 불필요한 데이터를 저장하지 않는다는 점이다. 그리고 브라우저마다 차이는 있지만 용량이 약 5MB로 매우 크다. 단점으로는 HTML5를 지원하지 않는 브라우저에서는 사용이 불가능하다.
    자동 로그인 기능이나 다크모드 테마와 같은 사용자에게 귀속되면서 유출되어도 상관없는 정보들에는 로컬 스토리지를 사용

    세션스토리지의 가장 큰 특징은 브라우저 창을 닫는 순간 영구적으로 데이터가 삭제된다. 세션스토리지도 마찬가지로 서버에 불필요한 데이터를 저장하지 않고, 용량이 5MB로 매우 크고 HTML5를 지원하지 않는 브라우저에서는 사용할 수 없다.
    폼 정보나 장바구니 정보 등 휘발성이 강한 개인적인 내용에는 세션 스토리지 사용

  8. 컴파일러와 인터프리터

    컴파일러는 전체 코드를 보고 명령어를 수집하고 재구성한다(C, C++등이 해당)

    인터프리터는 소스코드의 각 행을 연속적으로 분석하며 실행한다. 인터프리터는 고레벨 언어를 중간 레벨 언어로 변환한 뒤에 각 행마다 실행하기 때문에 보통은 컴파일러가 인터프리터보다 빠르다. Python, javascript 등이 해당하며, Java는 컴파일러에서 .class파일로 변환한 뒤에 인터프리터 과정으로 한번 더 해석을 하기 때문에 두가지 방법 모두 쓴다고 할 수 있다.

  9. 프로세스와 스레드

    프로세스는 운영체제로부터 CPU자원을 할당받는 작업의 단위이다. 메모리에 올라와서 실행되고 있는 프로그램의 인스턴스를 의미한다. Stack, Code, Data, Heap 구조로 되어 있다.

    스레드는 프로세스 내에서 실행되는 여러 흐름의 단위이다. 프로세스 내에서 각각 Stack만 따로 할당 받고 Code, Data, Heap 영역은 공유한다.
    멀티 스레드가 가지는 장점은 첫 번쨰로 프로세스를 생성하여 자원을 할당하는 시스템 콜이 감소하므로 자원을 효율적으로 관리할 수 있고, 스레드 간의 통신 비용이 더 적게 발생한다.
    대신, 스레드 간에 공유자원으로 인한 문제를 위해 동기화에 대해 신경을 써줘야 한다.

  10. 웹 표준과 웹 호환성

    웹표준(WEB Standards)이란 브라우저 종류 및 버전에 따른 기능 차이에 대하여 호환이 가능하도록 제시된 표준으로, 다른 기종 혹은 플랫폼에 따라 달리 구현되는 기술을 동일하게 구현함과 동시에 어느 한쪽에 최적화되어 치우치지 않도록 공통요소를 사용하여 웹 페이지를 제작
    W3C(World Wide Consortium)가 권고한 표준안에 따라 웹사이트를 작성할 때 이용하는 HTML, CSS, JavaScript 등에 대한 규정을 담고 있으며 웹 표준의 궁극적인 목적은 웹사이트에 접속한 사용자가 어떠한 운영체제나 브라우저를 사용하더라도 동일한 결과를 보이게 하는 것이다.

    웹호환성(Cross Browsing)은 표준 웹 기술을 사용하여 운영체제, 브라우저 등 어느 한쪽으로 최적화되거나 종속되지 않도록 공통 요소를 사용하여 웹 페이지를 제작하는 기법으로 웹 사이트 사용 시 운영체제 및 브라우저 간 동일한 결과가 나오도록 의미하는 웹 상호운용성의 개념에 웹 표준의 준수를 포함하는 개념이다.

  11. 웹 접근성

    웹 접근성(Web Accessibility)란 장애인, 고령자 등 모든 사용자가 어떠한 기술이나 환경에서도 전문적인 능력 없이 웹사이트에서 제공하는 모든 정보를 동등하게 접근하고 이해할 수 있는 권리입니다.
    img alt태그 추가(이미지의 설명), header태그를 사용해 문서 정리, label태그 적용, skip메뉴, 탭 인덱싱 가능(tab-indexable) ...

  12. UI와 UX

    UI(User Interface) : 사용자가 앱을 사용할 떄 마주하는 디자인, 레이아웃, 기술적인 부분
    디자인의 구성 요소인 폰트, 색깔, 줄간격 등 상세한 요소가 포함되고, 기술적 부분은 반응형이나 애니메이션 효과 등이 포함된다. 따라서 UI는 사용자가 사용할 떄 큰 불편함이 없어야 하며, 만족도를 높여야 한다.

    UX(User Experience) : 앱을 주로 사용하는 사용자들의 경험을 분석하여 더 편하고 효율적인 방향으로 프로세스가 진행될 수 있도록 만드는 것이다.(터치화면, 사용자의 선택flow 등) UX는 통계자료, 데이터를 기반으로 앱을 사용하는 유저들의 특성을 분석하여 상황과 시점에 맞도록 변화시킬 수 있어야 한다.

  13. 반응형(Response Website), 적응형(Adaptive Website)

    반응형 웹 디자인이란 하나의 웹 사이트에서 PC, 스마트폰, 태블릿 등 접속하는 디스플레이의 종류에 따라 화면의 크기가 자동으로 변하도록 만든 웹페이지 접근 기법
    웹사이트를 PC용과 모바일용으로 별개로 제작하지 않고, 하나의 공용 웹사이트를 만들어 다양한 디바이스에 대응
    PC용 url과 모바일용 url이 동일하기 때문에 검색 포털 등 광고를 통한 사용자 접속을 효율적으로 관리할 수 있다. 또한, 웹 페이지 내용 수정 시 하나의 페이지만 수정하면 모든 디바이스에서 동일하게 반영

    적응형 웹 디자인이란 서버나 클라이언트에서 웹에 접근한 디바이스를 체크하여 그 디바이스에 최적화된 미리 정해 놓은 각 디바이스의 디스플레이에 맞는 웹을 보여주는 디자인을 의미

  14. SPA(Single Page Application)

    SSR(Server Side Rendering) : 데이터가 많을수록 성능문제(요청할 때마다 새로고침이 일어나면서 페이지 로딩), 검색엔진 최적화의 장점
    CSR(Client Side Rendering) : 사용자의 행동에 따라 필요한 부분만 다시 읽어온다. 따라서 서버 측에서 렌더링하여 전체 페이지를 다시 읽어 들이는 것보다 빠른 인터랙션을 기대, 검색엔진 수집에 어려움
    CSR에는 모바일 시대에 들어서 SPA등장
    SPA(Single Page Application) : 서버로부터 완전한 새로운 페이지를 불러오지 않고 현재의 페이지를 동적으로 다시 작성함으로써 사용자와 소통하는 웹 애플리케이션이나 웹사이트를 말한다.

  15. HTML, CSS, JavaScript

    HTML은 HyperText Markup Language의 줄임말로, 웹 문서를 만드는 언어입니다
    HyperText는 문서를 서로 연결해주는 링크를 의미하고, Markup은 표시하다는 뜻으로 HTML은 문서의 위치 링크를 표시하는 언어입니다.
    CSS는 Cascading StyleSheet의 줄임말로, 웹 문서를 꾸미는 언어입니다.
    JavaScript는 웹페이지에 동적인 요소를 넣을 수 있는 객체 기반의 스크립트 프로그래밍 언어로, ECMAScript는 JavaScript표준화를 위하여 만들어진 언어입니다. ES6는 ECMAScript의 6번째 버전입니다.

  16. 트리거

    데이터베이스 트리거(Database Trigger)는 테이블에 대한 이벤트에 반응해 자동으로 실행되는 작업을 의미한다.
    자동으로 실행되도록 정의된 저장 프로시저(insert, update, delete 문에 대한 응답을 자동으로 호출)
    업무 규칙 보장, 업무 처리 자동화, 데이터 무결성 강화

  17. 정규표현식(regular expression)

    정규식은 특정한 규칙을 가진 문자열의 집합을 표현하는 데 사용하는 형식 언어이다. 정규 표현식은 많은 텍스트 편집기와 프로그래밍 언어에서 문자열의 검색과 치환을 위해 지원하고 있다.

  18. 중앙처리장치(CPU : Central Processing Unit)

    컴퓨터 시스템 전체의 작동을 통제하고 프로그램의 모든 연산을 수행하는 핵심적인 장치이다.
    작업 감독, 산술 연산, 논리 연산, 프로그램/데이터 저장 등의 기능을 수행한다.
    제어장치, 연산장치, 레지스터(주기억장치)로 구성되며, 버스를 통해 이들을 연결하여 데이터를 주고받는다.

  19. 운영체제

    컴퓨터 시스템을 구성하고 있는 하드웨어 장치와 일반 컴퓨터 사용자 또는 컴퓨터에서 실행되는 응용프로그램의 중간에 위치하여 사용자들이 보다 쉽고 간편하게 컴퓨터 시스템을 이용할 수 있도록 제어 관리하는 프로그램이다.
    중앙처리장치의 시간, 메모리 공간, 파일 기억장치 등의 자원들을 효율적으로 관리(DOS,UNIX, Linux, Windows)

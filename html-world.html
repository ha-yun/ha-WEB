HTML 구조

  • HTMl문서는 일반 문서와 달리 정해진 형식에 맞추어 내용을 입력해야 한다.
  • 웹 문서는 보통 !DOCTYPE html로 시작해 html, head, body라는 3개의 영역으로 구성되어 있다.
    1. !DOCTYPE html : 현재 문서가 HTML5언어로 작성한 웹 문서라는 뜻이다.
    2. html : 웹 문서의 시작과 끝을 나타내는 태그이다, 웹 브라우저가 html태그를 만나면 /html까지 소스를 읽어 화면에 표시한다.
    3. head : 웹 브라우저가 웹 문서를 해석하는데 필요한 정보를 입력하는 부분이다.
    4. body : 실제로 웹 브라우저 화면에 나타나는 내용
  • html태그에서 lang 속성으로 문서에서 사용할 언어를 지정할 수 있다, 검색 사이트에서 특정 언어로 제한해 검색할 때 필요,
    화면 낭독기에서 웹 문서를 소리 내어 읽어줄 때 다르게 할 수 있다.
  • head태그에는 meta, title 태그를 사용
    • meta는 데이터에 관한 데이터
    • meta charset='UTF-8' : 웹 서버는 영어가 기본이므로 화면에 한글로 된 내용을 표시할 때는
      UTF-8이라는 문자 세트를 사용한다고 웹 브라우저에 알려줘야 한다.
    • title은 웹 문서의 제목을 나타낸다.

  • 시맨틱 태그

    • HTML의 태그는 그 이름만 봐도 의미를 알 수 있어 시맨틱(semantic, 의미론적인)태그라고 한다.
    • 시맨틱 태그를 사용하면 웹 브라우저가 HTML의 소스 코드만 보고도 제목과 본문 메뉴를 쉽게 알 수 있다.
      또한, 문서 구조가 정확히 나눠지므로 PC나 모바일의 웹 브라우저와 여러 스마트 기기의 다양한 화면에서 웹 문서를 표현하기 쉽다.
      마지막으로, 인터넷에서 웹 사이트를 검색할 대 필요한 내용을 정확히 찾을 수 있다.
    • 헤더 영역을 나타내는 header, 내비게이션 영역을 나타내는 nav, 핵심 콘텐츠를 담는 main, 독립적인 콘텐츠를 담는 article,
      콘텐츠 영역을 나타내는 section,사이드 바 영역을 나타내는 aside, 푸터영역을 나타내는 footer, 여러 소스를 묶는 div
  • html은 문서의 요소요소를 구분, css는 화면상에서 적절한 곳에 있도록 구분
  • 같은 파일에 두 언어를 사용하는 것은 읽는데 피로감,, 따로 작성하도록 한다!

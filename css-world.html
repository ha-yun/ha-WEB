CSS박스 모델

  • 블록레벨 요소란 태그를 사용해 요소를 삽입했을 때 혼자 한 줄을 차지하는 것을 가리킨다.
  • 인라인레벨 요소는 한 줄을 차지 하지 않는다, 콘텐츠만큼만 영역을 차지하고 나머지 공간에는 다른 요소가 올 수 있다.
  • 웹 문서의 블록 레벨 요소는 모두 박스 형태로, 이렇게 박스 형태인 요소를 박스 모델 요소라고 한다.
  • 박스 모델은 콘텐츠 영역, 박스와 콘텐츠 영역 사이의 여백인 패딩, 박스의 테두리, 박스 모델 사이의 여백인 마진 등의 요소로 구성
  • 상자의 외부넓이부터 Margin, Border, Padding, Content
  • box-sizing은 박스 모델의 너비와 높이를 어떻게 결정할 것인지에 따라 border-box, content-box중에서 선택할 수 있다.
    - border-box : 테두리까지 포함해서 너빗값을 지정
    - content-box : 콘텐츠 영역만 너빗값을 지정(기본값)
  • 박스 모델의 방향은 맨위부터 시작해서 top > right > bottom > left처럼 시계방향의 순서이다.
  • display속성을 사용하면 블록 레벨 요소와 인라인 레벨 요소를 서로 바꿔서 사용할 수 있다.
    block, inline, inline-block(inline과 block요소를 모두 가지고 있으며 마진과 패딩을 지정), none(해당 요소를 화면에 표시x)
  • float속성은 웹 요소를 문서 위에 떠 있게 만든다
    left, right, none(좌우 어디에도 배치x, 기본값)
  • clear속성으로 float속성을 해제
    right, left, both
  • left, right, top, bottom 속성을 이용하여 웹 요소의 취지를 정할 수 있다.

  • animation 속성은 특정 지점에서 스타일을 바꾸면서 애니메이션을 만드는데, 이렇게 애니메이션 중간에 스타일이 바뀌는 지점을 키프레임keyframe이라고 한다.

Responsive

  1. meta 태그 - viewport
    2.
    • viewport란 웹페이지에서 사용자의 보이는 영역(visible area)를 말한다.
    • viewport를 지정하면 접속한 기기의 화면에 맞추어 확대하거나 축소해서 표시할 수 있다.
    • viewport는 head태그에 작성
      meta name="viewport" content=""
    • content 안에서 사용하는 속성
      - width : 뷰포트 너비, device-width 또는 크기 입력
      - height : 뷰포트 높이, device-height 또는 크기 입력
      - user-scalable : 확대축소 가능여부, yes or no
      - initial-scale : 초기 확대축소 값, 1~10
      - minimum-scale, maximum-scale : 뷰포트의 최소최대배율값(0~10)
    • 뷰포트 단위
      - vm : viewport width, 1vw는 뷰포트 너비의 1%와 같다.
      - vh : viewport height, 1vh는 뷰포트 높이의 1%와 같다.
      - vmin : viewport minimum, 뷰포트의 너비와 높이 중에서 작은 값의 1%와 같다.
      - vmax : viewport maximum, 뷰포트의 너비와 높이 중에서 큰 값의 1%와 같다.
  2. 미디어 쿼리
    3.
    • 미디어 쿼리media queries는 사이트에 접속하는 장치에 따라 특정한 css 스타일을 사용하는 방법
    • 미디어 쿼리를 사용하면 접속하는 기기의 화면 크기에 따라 레이아웃이 달라진다.
    • @media[only|not|and] 미디어 유형
    • 미디어 유형의 종류
      all, print, screen, tv
      aural(음성 합성 장치,화면을 읽어 소리로 출력해 주는 장치)
      braille(점자 표시 장치), handheld(패드처럼 손에 들고 다니는 장치)
      projection, tty(디스플레이 기능이 제한된 장치, px단위를 사용할 수 없다.), embossed(점자 프린터)
    • 뷰포트의 너비와 높이를 미디어의 쿼리의 조건으로 사용할 수 있다.
      width, height, min-width, max-width
    • 단말기 너비나 높이를 고려해 사용
      device-width, device-height, min-device-width, max-device-width
    • 화면 회전 속성
      orientation: portrait , 단말기의 세로 모드 지정
      orientation: landscape, 단말기의 가로 모드 지정
    • 미디어 쿼리를 작성할 때 화면 크기에 따라 서로 다른 CSS를 적용할 분기점을 중단점이라고 한다.
    • 시중에 나온 기기를 모두 반영할 수는 없으므로 모바일과 태블릿, 데스크톱 정도로만 구분하는 것이 좋다
    • 처리 속도나 화면 크기 등에서 다른 기기보다 모바일의 제약 조건이 더 많으므로 모바일의 레이아웃을 기본으로 CSS를 만든다
      그러고 나서 태블릿과 데스크톱에 맞춰 기능과 스타일을 추가(모바일 퍼스트 기법)
    • 스마트폰(세로minwidth - portrait 320px, 가로min-width - landscape 480px
      태블릿 : 세로 크기가 768px이상, 가로는 데스크톱과 똑같이 1024px이상으로 지정
      데스크톱 : 화면 크기가 1024px이상
    • 모바일 기기의 뷰포트 크깃값이 정리되어 있는 사이트
